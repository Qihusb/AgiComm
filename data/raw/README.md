# `data/raw/` 原始数据说明

本目录存放**未经加工**的画像与相关原始数据。下列说明按**当前目录内实际文件**整理，并补充**网民画像**维度的分类与爬取关键词约定（与媒体画像区分）。

---

## 数据类型区分


| 类型       | 说明                                                         |
| -------- | ---------------------------------------------------------- |
| **网民画像** | 面向普通网民/用户群体的画像数据；命名上常使用 `tech_*`、`culture_*` 等主题前缀（见下文分类）。 |
| **媒体画像** | 面向新闻媒体机构的画像；本仓库中对应 `**media_profiles.json`**。              |


---

## 目录内文件一览

### `media_profiles.json`

- **类型**：媒体画像（非网民画像）。
- **格式**：JSON 对象，顶层键为媒体 ID（如 `media_081`），值为该媒体的完整画像。
- **内容结构（概要）**：
  - `**static_profile`**：静态属性，如 `media_id`、`media_name`、`media_name_en`、`country`、`region`、`media_type`、`ownership_type`、`is_press_conference_regular`、`social_presence`、`website` 等。
  - `**behavior_dynamic**`：动态行为，常见块包括：
    - `**press_conference**`：发布会参与与议题、提问意图等（如 `topic_focus`、`question_intent`）。
    - `**news_reports**`：报道侧议题分布等。
    - 其他与一致性、社媒等相关的度量（以实际 JSON 字段为准）。
- **用途**：供下游脚本（如 `src/utils/media_processor.py`）扁平化、清洗后写入 `data/processed/` 等。

> 若仓库未附带完整原始集（体积或隐私原因），可自行按上述结构准备 JSON 并放入本目录。

## 网民画像：主题分类与爬取关键词

以下分类用于**网民画像**相关数据采集或文件命名时的语义约定；**与 `media_profiles.json` 无对应关系**。

### 科技（Tech）


| 标识 / 前缀            | 说明                      | 爬取关键词                                     |
| ------------------ | ----------------------- | ----------------------------------------- |
| **tech_ai**        | 人工智能方向                  | 人工智能                                      |
| **tech_data**      | 数据与创新生态等                | 全球创新指数、科技集群、新能源汽车、灯塔工厂、5G网络、数字丝路、知识产权、PCT |
| **tech_health**    | 健康与科技交叉（关键词约定与数据侧一致时使用） | 全球创新指数、科技集群、新能源汽车、灯塔工厂、5G网络、数字丝路、知识产权、PCT |
| **tech_aerospace** | 航空航天相关                  | （航空航天领域相关检索词，按具体采集方案补充）                   |


### 文化（Culture）


| 标识 / 前缀             | 说明         | 爬取关键词                                     |
| ------------------- | ---------- | ----------------------------------------- |
| **culture_defence** | 涉舆叙事与争议议题  | “巴黎奥运会药检针对”、“西方抵制冬奥会”、“旅美熊猫丫丫风波”、“流失文物追讨” |

## 使用建议

- 媒体实验链路：以 `**media_profiles.json`** 为入口，配合 `configs/media_schema.json` 与 `src/utils/` 下处理脚本。
- 网民画像链路：按上表主题与关键词组织原始数据；落地文件名可与团队约定（如按 `tech_*` / `culture_*` 分文件），并在本 README 中随仓库迭代补充**实际文件名**与字段说明。
