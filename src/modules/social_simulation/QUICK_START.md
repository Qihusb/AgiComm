### 使用流程

```
1️⃣  输入事件
    - 在"事件描述"文本框输入要仿真的事件
    - 示例: "人工智能发展带来新机遇和挑战"

2️⃣  调整情感和立场
    - 情感强度: 0=平淡 → 1=激情
    - 立场倾向: -1=反对 → 1=支持

3️⃣  选择仿真参数
    - 种子策略: 高影响力用户 / 高情感用户 / 随机
    - 种子数量: 1-20个初始用户
    - 最大步数: 5-50次传播迭代

4️⃣  点击"开始仿真"
    - 系统会加载数据、构建网络
    - 运行Message传播仿真
    - 计算情感/立场演化

5️⃣  查看结果
    - 📊 概览卡片: 总参与人数、覆盖率等
    - 📈 传播曲线: 每步增长情况
    - 😊 情感分析: 平均情感和波动
    - 🎯 立场分析: 平均立场和极化程度
    - 🌱 种子节点: 初始用户列表
```

## 📊 理解结果指标

### 概览卡片

| 指标           | 含义                   | 范围         |
| -------------- | ---------------------- | ------------ |
| **总参与用户** | 传播过程中激活的用户数 | 0-N          |
| **传播覆盖率** | 激活用户数/总用户数    | 0-1 (0-100%) |
| **传播深度**   | 信息传播的轮次数       | 1-max_steps  |
| **平均增长率** | 每步平均增长百分比     | 0-1          |

### 情感分析

| 指标         | 含义                 | 解读                             |
| ------------ | -------------------- | -------------------------------- |
| **平均情感** | 活跃用户的平均情感值 | > 0.7: 正向感染; < 0.3: 负向蔓延 |
| **情感波动** | 标准差               | 高: 群体分化; 低: 群体一致       |

### 立场分析

| 指标         | 含义               | 解读                           |
| ------------ | ------------------ | ------------------------------ |
| **平均立场** | 活跃用户的平均立场 | > 0: 偏向支持; < 0: 偏向反对   |
| **立场分化** | 极化程度           | 高: 意见严重分歧; 低: 观点一致 |

## 🐛 常见问题

### Q1: 后端无法启动？

**错误**: `ModuleNotFoundError: No module named 'networkx'`

**解决**:

```bash
pip install networkx pandas numpy fastapi uvicorn
```

### Q2: 前端无法连接后端？

**检查**:

1. 后端是否运行: `http://localhost:8001`
2. 前端环境变量: 检查 `frontend/.env` 中 `VITE_API_BASE=http://localhost:8001`
3. 跨域问题: 检查后端是否启用CORS

**解决**:

```bash
# 重启后端
python -m src.modules.api

# 重启前端
cd frontend && npm run dev
```

### Q3: 数据文件缺失？

**错误**: `FileNotFoundError: ...netizen_science_standardized_profiles.csv`

**解决**: 确保文件存在于

```
data/processed/netizen_science_standardized_profiles.csv
```

### Q4: 仿真很慢？

**原因**: 网络太大或步数太多

**解决**:

- 减少种子数量 (3-5)
- 减少最大步数 (5-10)
- 使用高影响力用户作为种子
